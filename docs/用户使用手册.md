# 用户使用手册 (User Guide)

欢迎使用 **PDF转Word自动监控工具**！本文档将指导您如何配置和使用该工具，以及如何将其作为亮点项目展示在您的简历中。

---

## 1. 快速开始

### 方式一：拖拽转换（推荐）
1.  双击运行桌面上的 **`启动PDF监控.bat`**。
2.  将您的 PDF 文件直接拖入 **`PDF转Word工具`** 文件夹中。
3.  程序会自动检测到新文件，并开始转换。
4.  转换完成后，同名 `.docx` 文件会出现在同一文件夹内。

### 方式二：后台静默运行
1.  双击运行 **`start_pdf_monitor.bat`**。
2.  程序将在后台运行（无黑色窗口），持续监控该文件夹。
3.  如需停止，请关闭终端窗口或在任务管理器中结束 `python` 进程。

---

## 2. 常见问题解答 (FAQ)

### Q1: 转换后的 Word 文件在哪里？
**A:** 默认情况下，生成的 Word 文件与原 PDF 文件在同一个文件夹内。

### Q2: 为什么有些 PDF 转换效果不好？
**A:** 本工具基于 `pdf2docx` 库，对于标准文字版 PDF 效果极佳。但对于扫描版（即全是图片的 PDF），转换效果可能受限，因为需要 OCR（光学字符识别）功能支持，而本项目主要关注结构化文档转换。

### Q3: 遇到 "乱码" 或 "闪退" 怎么办？
**A:** 
-   **乱码**：通常是因为 Windows 终端编码（GBK vs UTF-8）冲突。我们已提供全英文版的上传脚本以避免此问题。
-   **闪退**：如果是上传脚本闪退，请确保您已连接互联网，且没有被防火墙拦截。如果是监控程序闪退，请查看目录下的 `pdf_monitor.log` 日志文件获取错误详情。

## 3. 开发者指南

如果您希望修改代码或进行二次开发：

1.  **环境配置**：
    ```bash
    pip install -r requirements.txt
    ```

2.  **运行测试**：
    我们在 `tests/` 目录下提供了测试脚本，运行以下命令验证环境：
    ```bash
    python tests/test_pdf_creator.py
    ```

3.  **核心代码**：
    主要逻辑位于 `pdf_monitor.py`，其中 `PDFHandler` 类负责事件处理，`ThreadPoolExecutor` 负责并发转换。

## 4. 关于简历与面试

本项目是一个非常优秀的 **"自动化工具/效率工程"** 案例。在面试中，您可以重点介绍以下技术点：

*   **事件驱动 (Event-Driven)**：使用 `watchdog` 监听文件系统事件，而不是低效的轮询。
*   **并发编程 (Concurrency)**：使用 `ThreadPoolExecutor` 线程池处理耗时的 I/O 操作，防止主线程阻塞。
*   **工程化思维**：包含完整的日志记录、异常处理、配置分离和自动化脚本。
